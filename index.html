____Bài 4: JavaScript Syntax (cú pháp)____

JavaScript Values

Cú pháp JavaScript định nghĩa 2 loại kiểu giá trị:

- Literals Không thay đổi giá trị

- Variables Thay đổi giá trị

JavaScript Literals (Không thay đổi giá trị)

- ví dụ: Nếu bạn khởi tạo const thì giá trị của nó sẽ không thay đổi khi bạn gán lại giá trị đó.

Các quy tắc cú pháp quan trọng nhất cho literals là:

Numbers được viết có hoặc không có decimals (số thập phân)

Strings là chữ được viết trong dấu nháy kép hoặc nháy đơn

JavaScript Variables (Thay đổi giá trị)

- ví dụ: Nếu bạn khởi tạo let thì giá trị của nó sẽ thay đổi khi bạn gán lại giá trị đó.

Variables là nơi chứa các giá trị dữ liệu. Phải được xác định bằng những tên duy nhất

JavaScript Identifiers (Thay đổi giá trị)

Một identifier là tên bạn đặt cho một variable.

Quy tắc nhận dạng:

- Phải được bắt đầu bằng dấu: _ or $

- Có thể chứa các chữ số sau ký tự đầu tiên

- Không thể là từ khoá dành riêng (let, const, if ..v..v..)

- Phân biệt chữ hoa chữ thường

JavaScript Operators (Toán tử)

JavaScript toán tử gán = là gán giá trị cho variables

JavaScript sử dụng số học toán tử + _ * / để tính toán các giá trị

JavaScript Expressions (Biểu thức)

Biểu thức là sự kết hợp của các giá trị variables và operators để tính toán thành một gía trị.

Biểu thức cũng có thể bao gồm variable (ví dụ: x * 10 hoặc "John" + " " + "Doe")

JavaScript is Case Sensitives (phân biệt chữ hoa chữ thường)

JavaScript identifiers là phân biệt chữ hoa chữ thường

Variables lastName và lastname là các variables khác nhau

JavaScript and Camel Case

Trong lịch sử, các lập trình viên đã sử dụng nhiều cách khác nhau để nối nhiều từ thành một tên biến:

Hyphens (dấu gạch nối):

first-name, last-name, master-card, inter-city.

Lưu ý:

Dấu gạch nối không được phép trong JavaScript. Chúng được dành riêng cho toán tử phép trừ.

Underscore(dấu gạch dưới):

first_name, last_name, master_card, inter_city.

Upper Camel Case (Pascal Case):(viết hoa chữ cái đầu tiên của các từ):

FirstName, LastName, MasterCard, InterCity.

Lower Camel Case:(viết thường chữ cái đầu tiên của từ, còn lại các chữ cái đầu tiên của các từ là viết hoa):

firstName, lastName, masterCard, interCity.

Lưu ý:

Các lập trình viên JavaScript có xu hướng sử dụng Lower Camel Case.

____Bài 5: JavaScript Statements (câu lệnh)____

JavaScript Programs

Một chương trình máy tính là một danh sách "instructions (hướng dẫn)" được máy tính "executed (thược thi)".

Những hướng dẫn lập trình này được gọi là statements (câu lệnh).

Các câu lệnh được thực thi từng câu một theo đúng thứ tự như chúng được viết.

Lưu ý:

Trong HTML, các chương trình JavaScript được thực thi bởi browser

JavaScript Statements

Câu lệnh JavaScript bao gồm: Values, Operators, Expressions, Keywords, and Comments

Lưu ý:

JavaScript programs (và JavaScript statements) thông thường được gọi là JavaScript code

Semicolons (dấu chấm phẩy) ;

Dấu chấm phẩy phân cách các câu lệnh JavaScript.

Thêm dấu chấm phẩy vào cuối mỗi câu lệnh thực thi.

Khi được phân tách bằng dấu chấm phẩy, được phép có nhiều câu lệnh trên một dòng.

Lưu ý:

Trên website, bạn có thể thấy các ví dụ không có dấu chấm phẩy.

Kết thúc câu lệnh với dấu chấm phẩy là không bẳt buộc, nhưng rất được khuyến khích.

JavaScript White Space

JavaScript bỏ qua nhiều khoảng trắng. Bạn có thể thêm dấu khoảng trắng vào tệp lệnh của mình để dễ đọc hơn.

Một cách thực hành tốt lè nên đặt khoảng trắng vào xung quanh toán tử ( = + - * / )

JavaScript Line Length and Line Breaks (Độ dài của dòng và ngắt dòng)

Để dễ đọc nhất, các lập trình viên thường tránh các dòng mã dài hơn 80 ký tự.

Nếu một câu lệnh JavaScript dài quá 80 ký tự trên một dòng, nơi tốt nhất để dừng là sau một toán tử.

JavaScript Code Blocks (Khối mã trong ngoặc nhọn {})

Câu lệnh JavaScript có thể nhóm lại với nhau trong các khối mã, bên trong dấu ngoặc nhọn {...}

Lưu ý:

Trong JavaScript với các câu lệnh được viết bên trong khối mã, nên sử dụng thêm hai dấu cách thụt đầu dòng cho khối mã đó.

JavaScript Keywords

Các câu lệnh JavaScript thường được bắt dầu bằng một keyword để xác định hành động JavaScript sẽ được thực hiện.

Dưới đây là một số keyword mà chúng ta tìm hiểu trong bài này:

var: Khai báo một variable

let: Khai báo một variable khối

const: Khai báo một khối không thay đổi giá trị

if: Đánh dấu một khối câu lệnh sẽ được thực thi theo một điều kiện

switch: Đánh dấu một khối câu lệnh sẽ được thực thi trong các trường hợp khác nhau

for: Đánh dấu một khối câu lệnh sẽ được thực thi trong vòng lặp

function: khai báo một function (hàm/chức năng)

return: Thoát khỏi một function

try: Thực hiện xử lý lỗi đối với một khối câu lệnh

____Bài 6: JavaScript Variables (Biến = vùng chứa dữ liệu)____

JavaScript Variables

Variables = Data Containers (Vùng chứa dữ liệu)

javascript variables là nơi chứa dữ liệu

JavaScript variables có thể được khai báo theo 4 cách:

Modern JavaScript

- Sử dụng let

- Sử dụng const

Older JavaScript

- Sử dụng var (Không được đề xuất)

- Tự động (gán giá trị nhưng không khai báo) (Không được đề xuất)

Lưu ý:

Các biến là labels cho các giá trị.

Biến là nơi chứa dữ liệu

JavaScript Identifiers

Các biến được xác định bằng unique names (tên duy nhất) được gọi là identifier (định danh).

Tên có thể đặt ngắn như x, y, z.

Tên có thể mang tính mô tả age, sum, carName.

Nguyên tắc để xây dựng tên indentifier là:

- Tên có thể chứa các chữ cái, chữ số, dấu gạch dưới và ký hiệu dollar.

- Tên phải bắt đầu bằng một chữ cái, dấu $ hoặc dấu gạch dưới (_).

- Tên có phân biệt chữ hoa, chữ thường (X (chữ hoa) khác với x (chữ thường))

- Các từ khoá dành riêng cho JavaScript không thể được sử dụng làm tên: (ví dụ: const, let, var, if, for, switch, ...v..v..)

Lưu ý:

Số không được phép làm ký tự đầu tiên trong tên.

Bằng cách này JavaScript có thể dễ dàng phân biệt identifier từ những con số.

JavaScript Underscore (_)

JavaScript coi dấu gạch dưới là một chữ cái.

identifier chứa _ là tên biến hợp lệ.

Một quy ước giữa các lập trình viên chuyên nghiệp là bắt đầu tên bằng dấu gạch dưới cho các biến là private

JavaScript Dollar Sign $

JavaScript cũng coi ký hiệu dollar là một chữ cái.

identifier chứa $ là tên biến hợp lệ.

Sử dụng $ không phổ biến lắm trong JavaScript, nhưng các lập trình viên chuyên nghiệp thường sử dụng nó làm bí danh cho hàm main trong thư viện JavaScript.

Declaring JavaScript Variables

Tạo một biến trong JavaScript được gọi là khai báo môt biến.

Khai báo một biến JavaScript bằng từ khoá let hoặc từ khoá const.

Declaring a Variable Using const

Luôn luôn sử dụng const Nếu các giá trị không thay đổi.

Đó là một cách lập trình tốt để khai báo tất cả các biến ở đầu tệp lệnh.

Declaring a Variable Using var

Từ khoá var đã được sử dụng trong tất cả các mã JavaScript trước năm 2015.

Từ khoá let và constlà từ khoá mới được sử dụng đối với JavaScript vào năm 2015.

Lưu ý:

Khi nào nên sử dụng var, let hoặc const?

Đó là một cách lập trình tốt để khai báo tất cả các biến ở đầu tệp lệnh.

1. Luôn khai báo biến.

2. Luôn sử dụng const nếu giá trị khởi tạo không thay đổi.

3. Luôn sử dụng const nếu không thay đổi đối với (Array và Objects).

4. Chỉ sử dụng let nếu không thể sử dụng const.

5. Không bao giờ sử dụng var nếu có thể sử dụng let hoặc const.

JavaScript Data Types

Các biến JavaScript có thể chứa 8 datatypes (kiểu dữ liệu), nhưng hiện tại hãy chỉ nghĩ đến numbers và strings.

Strings là văn bản được viết trong nháy kép hoặc nháy đơn.

Numbbers được viết không có dấu nháy kép.

Nếu đặt số trong nháy kép hoặc nháy đơn, số đó sẽ được coi là string text.

One Statement, Many Variables

Bạn có thể khai báo nhiều biến trong một câu lệnh.

Bắt đầu câu lệnh với let hoặc const và phân tách các biến bằng dấu phẩy:

Ví dụ:

let person = "JavaScript", carName = "Hello", price = 20;

Một khai báo có thể trải rộng trên nhiều dòng:

Ví dụ:

let person = "JavaScript",

carName = "Hello",

The Assignment Operator

Trong JavaScript, dấu bằng ( = ) là toán tử gán, không phải toán tử bằng.

Điều này khác với đại số. Những điều sau đây không có ý nghĩa với đại số:

x = x + 5

Trong JavaScript, tuy nhiên, nó hoàn toàn hợp lý: nó gán giá trị của x + 5 cho x.

(Nó tính giá trị của x + 5 và đặt kết quả vào x. Giá trị của x được tăng thêm 5).

Lưu ý:

Toán tử bằng được viết như sau == trong JavaScript

JavaScript Arithmetic

Giống như đại số, bạn có thể thực hiện phép tính số học với các biến JavaScript bằng cách sử dụng các toán tử như = và +:

Ví dụ:

let x = 5 + 2 + 3;

Bạn cũng có thể thêm chuỗi nhưng chuỗi sẽ được nối với nhau:

Ví dụ:

let x = "Hello" + " " + "Bạn"

Lưu ý:

Nếu bạn đặt một số trong dấu nháy kép, phần còn lại của có số sẽ được coi là strings, và được nối với nhau.

Ví dụ:

let x = "5" + 2 + 3

Kết quả của việc thêm "5" + 2 + 3 là: 523

Ví dụ:

let x = 2 + 3 + "5"

Kết quả của việc cộng 2 + 3 + "5" là: 55

____Bài 7: JavaScript Let____

JavaScript Let

Từ khoá let được giới thiệu trong ES6 (2015)

Các biến được khai báo bằng let có phạm vi khối.

Các biến được khai báo bằng let phải được khai báo trước khi sử dụng.

Các biến được khai báo bằng let không thể được khai báo lại trong cùng một phạm vi.

Block Scope

Trước ES6 (2015), JavaScript không có Block Scope

JavaScript đã có Global Scope và Function Scope.

ES6 đã giới thiệu hai từ khoá JavaScript mới: let và const.

Hai từ khoá này được cung cấp Block Scope trong JavaScript:

Ví dụ:

Các biến được khai báo trong khối {} không thể được truy cập từ bên ngoài khối:

{

let x = 2;

}

// x không thể được sủ dụng ở đây.

Global Scope

Các biến được khai báo bằng var luôn luôn có Global Scope.

Các biến được khái báo bằng var từ khoá KHÔNG thể có block scope:

Ví dụ:

{

var x = 2;

}

// x CÓ THỂ được sủ dụng ở đây.

Cannot be Redeclared

Các biến được xác định bằng let KHÔNG thể được khai báo lại.

Bạn không thể vô tình khai báo lại một biến được khai báo bằng let.

Với let bạn không thể làm điều này:

{

let x = "Hello Chang";

let x = 0;

}

Các biến được khái báo bằng varcó thểđược khai báo lại.

Với var bạn có thể làm điều này:

{

var x = "Hello Chang";

var x = 0;

}

Redeclaring Variables

Việc khai báo lại một biến bằng từ khoá var có thể gây ra vấn đề.

Khai báo lại một biến bên trong một khối cũng sẽ khai báo lại biến bên ngoài khối đó.

Ví dụ:

var x = 10;

// Ở đây x là 10.

{

var x = 2;

// Ở đây x là 2.

}

// Ở đây x là 2

Kết quả của việc khai báo lại x trong block scope var x = 2; cuối cùng là: 2

Khai báo lại một biến bằng từ khoá let có thể giải quyết bằng vấn đề này.

Khai báo lại một biến bên trong một khối sẽ không khai báo lại biến bên ngoài khối:

Ví dụ:

let x = 10;

// Ở đây x là 10.

{

let x = 2;

// Ở đây x là 2.

}

// Ở đây x là 10

Kết quả của việc khai báo lại x trong block scope let x = 2; cuối cùng là: 10

What is Good?

let và const có block scope.

let và const không thể khai báo lại.

let và const phải được khai báo trước khi sử dụng.

let và const không not bind với this.

let và const là not hoisted.

What is Not Good?

var không cần phải khai báo.

var là hoisted.

var binds to this.

Browser Support

Từ khoá let và const không được hỗ trợ trong Internet Explorer 11 trở về trước.

Bảng sau xác định các phiên bản trình duyệt đầu tiên được hỗ trợ đầy đủ:

Redeclaring

Việc khai báo lại một biến JavaScript bằng var được phép ở bất kỳ đâu trong chương trình:

Ví dụ:

var x = 2;

// Tại đây x là 2.

var x = 3;

// Tại đây x là 3.

Kết quả của việc khai báo lại var x = 3; cuối cùng là: 3

Với let KHÔNG được phép khai báo lại một biến trong cùng một khối:

Ví dụ:

var x = 2; // ĐƯỢC PHÉP

let x = 3; // KHÔNG ĐƯỢC PHÉP

{

let x = 2; // ĐƯỢC PHÉP

let x = 3; // KHÔNG ĐƯỢC PHÉP

}

{

let x = 2; // ĐƯỢC PHÉP

var x = 3; // KHÔNG ĐƯỢC PHÉP

}

Khi báo lại một biến với let, trong một khối khác, là được phép:

Ví dụ:

let x = 2; // ĐƯỢC PHÉP

{

let x = 3; // ĐƯỢC PHÉP

}

{

let x = 4; // ĐƯỢC PHÉP

}

Kết quả của việc khai báo lại var x = 3; cuối cùng là: 3

Let Hoisting

Các biến được định nghĩa bằng var được đưa lên trên cùng và có thể được khởi tạo bất kỳ lúc nào.

Ý nghĩa: Bạn có thể sử dụng biến trước khi nó được khai báo:

Ví dụ:

carName = "Hello"

var carName;

Các biến được định nghĩa bằngletcũng được đưa lên đầu khối, nhưng không được khởi tạo.

Ý nghĩa: Sử dụng biến let trước khi nó được khai báo sẽ dẫn đến kết quả ReferenceError:

Ví dụ:

carName = "Hello"

let carName;

____Bài 8: JavaScript Const____

JavaScript Const

Từ khoá const được giới thiệu tronng ES6 (2015)

Các biến được định nghĩa bằng const không thể được khai báo lại.

Các biến được định nghĩa bằng const không thể đươc gán lại.

Các biến được định nghĩa bằng const Có Block Scope

Cannot be Reassigned

Một biến được định nghĩa bằng từ khoá const không thể được gán lại:

Ví dụ:

const PI = 3.1454534;

PI = 3.14; // Điều này sẽ báo lỗi.

PI = PI + 10; // Điều này cũng sẽ báo lỗi.

Must be Assigned

Các biến const trong JavaScript phải được gán một giá trị khi chúng được khai báo:

Đúng

const PI = 3.14159265359;

Không đúng

const PI;

PI = 3.14159265359;

Khi nào nên sử dụng const trong JavaScript?

Luôn khai báo một biến với const khi bạn biết rằng không nên thay đổi giá trị:

Sử dụng const khi bạn khai báo:

- Một Array (mảng mới).

- Một Object (đôi tượng mới).

- Một Function (Chức năng mới).

- Một RegExp

Constant Objects and Arrays

Từ khoá const hơi gây hiểu nhầm:

Nó không định nghĩa một giá trị constant. Nó định nghĩa một constant reference (tham chiếu) tới một giá trị.

Vì điều này bạn KHÔNG thể:

- Gán lại một giá trị constant

- Tái chỉ định một mảng constant

- Tái chỉ định một đối tượng constant

Nhưng bạn CÓ THỂ:

- đổi các phần tử của mảng constant

- đổi các thuộc tính của đối tượngconstant

Constant Arrays

Bạn có thể thay đổi phần tử của một mảng constant:

Không đúng

// Bạn có thể tạo một mảng constant:

const cars = ["Window", "MacOS", "Linux"];

// Bạn có thể thay đổi một element:

cars[0] = "Web Site";

// Bạn có thể thêm một element:

cars.push("IOS");

Nhưng bạn không thể gán lại mảng:

Ví dụ:

const cars = ["Window", "MacOS", "Linux"];

cars = ["Hanoi", "Saigon", "Hue"]; // Error

Constant Objects

Bạn có thể thay đổi các thuộc tính của constant:

Ví dụ:

// Bạn có thể tạo một đối tượng constant:

const cars = {name:"what your name", age:30, address:"Hanoi"};

// Bạn có thể thay đổi property:

cars.age = 18;

// Bạn có thể thêm một property:

cars.color = "red";

Nhưng bạn KHÔNG thể gán lại đối tượng:

Ví dụ:

const cars = {name:"what your name", age:30, address:"Hanoi"};

cars = {name:"ZoRO", age:20, address:"China"}; // Error

Block Scope

Khai báo một biến với const cũng tương tự như let khi nói đến Block Scope.

Trong ví dụ này, x được khai báo trong khối không giống với x được khai báo bên ngoài khối:

Ví dụ:

const x = 10;

// Ở đây x là 10.

{

const x = 2;

// Ở đây x là 2.

}

// Ở đây x là 10.

Redeclaring

Khai báo lại một biến var hoặc let hiện có thành const, trong cùng phạm vi, điều đó là không được phép:

Ví dụ:

var x = 2; // Cho phép.

const x = 2; // Không được phép.

{

let x = 2; // Cho phép.

const x = 2; // Không được phép.

}

{

const x = 2; // Cho phép.

const x = 2; // Không được phép.

}

Gán lại một biến const hiện có, trong cùng phạm vi, điều đó là không được phép:

Ví dụ:

const x = 2; // Cho phép.

x = 2; // Không được phép.

var x = 2; // Không được phép.

let x = 2; // Không được phép.

const x = 2; // Không được phép.

{

const x = 2; // Cho phép.

x = 2; // Không được phép.

var x = 2; // Không được phép.

let x = 2; // Không được phép.

const x = 2; // Không được phép.

}

Khai báo lại một biến bằng const trong phạm vi khác, hoặc ở khối khác, điều đó là được phép.

Ví dụ:

const x = 2; // Được phép.

{

const x = 3; // Được phép.

}

{

const x = 4; // Được phép.

}

Constant Hoisting

Các biến được định nghĩa bằng var được hoisted (được đưa lên sử dụng trước) và có thể khởi tạo bất cứ lúc nào.

Ý nghĩa: Bạn có thể sử dụng dùng biến trước khi nó được khai báo:

Ví dụ:

Sử dụng điều này vẫn OK:

name = "Jone";

var name;

Các biến được định nghĩa với const cũng là hoisted (được đưa lên sử dụng trước) lên đầu, nhưng nó không được khởi tạo.

Ý nghĩa: sử dụng biến const trước khi nó được khai báo sẽ dẫn đến lỗi ReferenceError:

Ví dụ:

alert (name);

const name = "Jone";
